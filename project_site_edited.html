01

연구 선정 이유

한국의 연간 최대전력은 매년 7~8월 오후 발생. 도심 냉방 부하 완화 수단으로 ESS(에너지저장장치)가 주목받지만, "어디에 얼마짜리"는 도시 단위 정량 분석이 필요.

초기에 설정한 연구질문

RQ질문
RQ 1마곡동 건물별 전력 수요를 머신러닝으로 예측할 수 있는가?
RQ 2어느 시점·어느 건물이 피크 부하를 형성하는가?

RQ 4위 결과를 3D 디지털트윈으로 통합 시각화할 수 있는가?

처음에는 "건물 크기·층수·높이가 에너지를 결정한다"는 직관적 모델을 세우고 시작했습니다. 그런데 데이터를 직접 까보면서 그 가설이 무너졌고, 그 무너짐 자체가 이 프로젝트의 핵심 발견이 되었습니다.

전처리 핵심 결정들

03

핵심 발견

Finding 01
'음영도' 컬럼은 사실 자치구 코드였다
⚡

Moment: 음영도 고유값을 세어보니 정확히 25개

음영도로 알고 받았으나 분포는 11110~11740의 25개 고유값 — 서울시 25개 자치구 법정동코드. 11110 = 종로구, 11140 = 중구, ..., 11500 = 강서구(마곡동 소속). 받은 자료가 마곡동만이 아니라 서울시 전체 건물 데이터였음.

→ 영향: 강서구(11500)로 1차 필터 → '마곡' 키워드 건물의 좌표 분포(1~99분위)로 경계 근사 → 약 11,358동의 마곡 데이터 확보. 행정경계가 아닌 통계적 근사로, 한계로 명시.
Finding 02
건물 특성으로는 에너지를 예측할 수 없다
⚡

Moment: 면적 ~ 에너지 상관 r = 0.07

"큰 건물 = 많이 씀" 가설이 무너짐.

변수상관해석
건물면적r = 0.07사실상 무관
층수r = 0.03사실상 무관
건물높이r = 0.03사실상 무관
→ 영향: ML 입력을 "건물 특성" → "기상"으로 전환. 목표도 "개별 건물 예측" → "도시 단위 계절 예측"으로 재정의.
Finding 03
에너지는 사실상 '기온(계절)' 하나로 설명된다
⚡

Moment: 기온 ~ 에너지 상관 r = 0.70 (압도적)

월별 평균: 1월 17.8 → 6월 143.3 → 12월 20.9. 여름 단봉형.

기상 변수상관

강수r = 0.59
습도r = 0.40
일조r ≈ 0.06

→ 영향: 여름 단봉형 = 전기 위주(난방=가스 누락). ESS도 냉방 피크 대응에 초점.
Finding 04
강수가 1위 변수로 나왔지만, 그건 다중공선성 신호였다
⚡

Moment: 기상 변수 간 상관 0.79~0.89 (다중공선성)

XGBoost 첫 결과 강수 1위 → 비현실적. 변수 간 상관 0.8↑ → 기온·습도·강수 모두 "여름" 신호의 중복.

기온습도강수에너지
기온1.000.790.850.70
습도0.791.000.890.40
강수0.850.891.000.59
→ 영향: 강수·습도 제외 후 기온 단독 R² 동일. 최종 변수 CDD + HDD + 면적으로 단순화.

"가장 단순한 모델이 가장 정직했고, 가장 정직한 모델이 가장 잘 일반화됐다."

분석 방법론

알고리즘 선택보다 평가 설계가 결과의 신뢰도를 결정. 파이프라인: 병합 → 탐색 → ML 예측(3가지 분할 비교) → 군집화(K-Means) → ESS 시뮬레이션 → 3D 디지털트윈.

평가 설계 — 같은 모델, 다른 분할

분할 방식의미R²판단
건물단위 5-fold처음 보는 '건물' 예측력0.691정직한 평가
무작위 5-fold모든 행 무작위 분할0.701과대추정
월단위 분할처음 보는 '계절' 예측력0.054 (±0.88)시계열 일반화 불가

무작위 분할은 같은 달 데이터가 train·test 양쪽에 섞여 "월 평균 암기"가 일어난 과대추정입니다. 세 결과를 함께 봐야 "건물 예측은 되지만 시계열 일반화는 못한다"는 정직한 결론이 나옵니다.

건물 군집화

총량 기준으로 묶으면 "큰 건물 vs 작은 건물"만 갈립니다. 패턴 모양(여름 집중도)을 보려면 각 건물의 12개월 에너지를 합=1로 정규화한 뒤 K-Means를 적용합니다.

k 결정 — 실루엣 vs 해석

kInertiaSilhouette해석
210.50.616"여름형/덜여름형" — 너무 단순
37.40.466집중도 3단계로 해석 명확 (채택)
45.30.517국소최대값이나 해석 모호
54.20.447과분할

실루엣은 k=2가 가장 높지만, 데이터 자체가 단봉형 단일 패턴이라 둘로 가르기 쉽기 때문. 해석 가능성을 위해 k=3 채택.

예상과 달랐던 점

  • 질적 유형 차이(여름형/겨울형/평탄형) 기대 → 실제는 모두 여름 단봉형, 강도만 다름.
  • 겨울 비중 모든 군집 0.04~0.11로 매우 낮음 → 난방 피크 부재.

3D 디지털트윈

분석 결과를 실제 마곡 지도 위 인터랙티브 시뮬레이션으로 통합.

기술 스택 선택

후보장점단점채택
Pydeck파이썬, 빠른 시작3D 표현 제한참고
Kepler.gl코딩 거의 없음커스텀 제한고려
CesiumJS + OSM Buildings실제 3D 건물, 지구본 위 표현토큰 필요, 학습곡선채택

좌표계 문제 — 추측을 사실로 교정

EPSG:5186 추정 → 엉뚱한 곳. 후보 5개 역변환 검증 → 정답은 EPSG:5179 (UTM-K).

lon0 = 127.5;  lat0 = 38
k0   = 0.9996; FE = 1000000;  FN = 2000000
ellipsoid = "GRS80"

색상 매칭 전략 — 격자 기반

OSM 건물과 1:1 매칭 불가 → 50m 격자로 나눠 위치 매칭. 전체 OSM 건물에 빈틈없이 색 부여. 단, 건물 단위가 아닌 격자 단위 근사라는 한계 명시.

디지털트윈 핵심 기능

📅 월 슬라이더 + 1년 재생

1~12월 슬라이더로 계절 변화 시각화. 각 건물의 그 달 실측 에너지가 색으로 표현.

🌡️ 기온 시나리오 (-5 ~ +10°C)

ML 회귀계수(CDD/HDD)를 JS에 심어 실시간 재계산. "온난화 +3°C 시" 마곡 피크 변화 즉시 확인.

🔋 ESS 설치 토글 + 시설 모델

버튼 클릭 시 마곡 중앙에 배터리 컨테이너 6개 + 관제타워 + 후광이 솟아오르는 ESS 단지 생성.

⚡ 충방전 이펙트 + 빛줄기

충전 시 파란빛 수축, 방전 시 노란빛 확장 + 주변 피크 건물 14개로 흐르는 빛줄기(PolylineGlow).

📊 실제 시뮬레이션 곡선 연동

용량 슬라이더(1~30%)가 실제 ESS 저감률 곡선(4.7%→38.5%)을 그대로 따름. 발표 숫자와 디지털트윈 숫자가 일치.

🎬 시네마틱 인트로

결론

  1. 마곡동은 냉방 의존형 에너지 구조로 여름 피크가 뚜렷하며, ML 분석(r=0.70)과 군집화에서 모두 일관되게 확인되었다.
  2. 마곡 중앙에 일에너지 10% 규모 공용 ESS를 두면 여름 피크의 약 21%를 완화할 수 있다. 비용 대비 효율의 변곡점은 10~20% 구간이다.
  3. 기온이 +3°C 오르면 같은 효과 유지에 약 7% 더 큰 ESS가 필요하다(10% → 10.7%). IPCC RCP 8.5 시나리오 대응 시 참고.
  4. 모든 결과는 3D 디지털트윈에서 월·기온·ESS를 직접 조절하며 실시간 검증 가능하다.
